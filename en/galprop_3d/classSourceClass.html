Detailed Description

Interface class for source classes in galprop.

Each source class provides injection spectrum for arbitrary types of CR particles. There is in general no limitations to the flexibility of source classes.

Galprop expects the units for the injection to be density per unit total momentum per unit time [cm^-3 MeV^-1 s^-1]. If specified, these are normalized such that the flux of protons at a specified energy at the solar location is a given number. All primary fluxes, apart from primary electrons and positrons, are adjusted with the same number. There is a special treatement for electrons and positrons, they are normalized such that the total electrons at a specified energy at the solar location is a given number.

Note that it is not possible to mix relative and absolute source classes. That would need the memory and CPU to store and propagate separately contributions from fixed and relative source classes.

Uses the Registry.h class to create instances. There is a reserved parameter called "SourceClassType" that gives the name of the class in the registry. This allows for easy extension of the capabilities of GALPROP, e.g. to hook it up with DM sources.

Add the time varying source to the particle.

The routine should add to the primary_source_function only. It is expected that this routine is called using the same particle for many different source classes.

This should be in addition to any specified steady state source. Currently only called for timestep == 2 and 3D. time is in years.
